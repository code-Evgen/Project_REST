Pet-project - Meteo_REST_collector

REST API service collecting weather data from sensors.
Directories:
- project3 - REST API service
- project_3-client - sensor example App

REST API service - Spring REST APP allowing
	- register new sensor
	- receive data from sensor
	- get all measurements from DB
	- get amount of raining days

Sensor example App - java app allowing:
	- send request for registering in REST API service
	- send 1000 example measurements (temperature, raining day) to REST API service
	- get saved measurements from REST API service and displaying with a chart

===================================================

Database structure:
- table "sensor"

	CREATE TABLE IF NOT EXISTS public.sensor
	(
		id integer NOT NULL GENERATED BY DEFAULT AS IDENTITY ( INCREMENT 1 START 1 MINVALUE 1 MAXVALUE 2147483647 CACHE 1 ),
		name character varying COLLATE pg_catalog."default" NOT NULL,
		CONSTRAINT uniq_id UNIQUE (id)
	)


-table "measurement"

	CREATE TABLE IF NOT EXISTS public.measurement
	(
		id integer NOT NULL GENERATED BY DEFAULT AS IDENTITY ( INCREMENT 1 START 1 MINVALUE 1 MAXVALUE 2147483647 CACHE 1 ),
		value double precision NOT NULL,
		raining boolean NOT NULL DEFAULT false,
		date_time timestamp without time zone NOT NULL,
		sensor_id integer NOT NULL,
		CONSTRAINT measurement_sensor_id_fkey FOREIGN KEY (sensor_id)
			REFERENCES public.sensor (id) MATCH SIMPLE
			ON UPDATE NO ACTION
			ON DELETE NO ACTION
	)
